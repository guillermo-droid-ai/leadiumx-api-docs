Make Outbound Voice Call
POST /make-outbound-voice-call

Initiate an outbound voice call using AI agents. Calls are powered by Retell AI with dynamic variable support for personalization.

Parameters
Parameter Type Description
agent_id string Required - The Retell AI agent ID
from_number string Required - From number (E.164 format: +1XXXXXXXXXX)
to_number string Required - Prospect phone number (E.164 format)
agent_version integer Optional - Agent version (default: 0)
dynamic_vars_values object Optional - Custom variables for agent context
Request
curl -X POST https://api.leadiumx.com/make-outbound-voice-call \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "agent_id": "agent_67147b301907688dfbb1404888", "from_number": "+17122183438", "to_number": "+18055551234", "agent_version": 7, "dynamic_vars_values": { "first_name": "John", "address": "123 Main St, Miami FL", "reason_for_selling": "Need cash quickly", "ballpark_price": "$350,000" } }'
Response
✓ Success (201)
{ "success": true, "call_id": "call_7f865013ec64f6d498b70b7ce6d", "agent_id": "agent_67147b301907688dfbb1404888", "from_number": "+17122183438", "to_number": "+18055551234", "status": "initiated", "logged_to_db": true, "timestamp": "2026-04-18T18:05:00.000Z" }

Send SMS Message
POST /send-initial-sms

Send SMS messages to prospects with optional AI context.

Parameters
Parameter Type Description
phone_number string Required - Recipient phone (E.164 format)
message string Required - SMS message content
agent_id string Optional - Associated agent ID
dynamic_vars_values object Optional - Custom context variables
Request
curl -X POST https://api.leadiumx.com/send-initial-sms \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "phone_number": "+18055551234", "message": "Hi John! We have a cash offer for your Miami property. Reply INTERESTED for details.", "agent_id": "agent_67147b301907688dfbb1404888" }'
Response
✓ Success (201)
{ "success": true, "message_id": "SMS-1713450600000", "to_number": "+18055551234", "status": "sent", "timestamp": "2026-04-18T18:05:00.000Z" }

Error Handling

The API uses standard HTTP status codes. Error responses include a message field with details.

Status Code Meaning Example
200 Success Request processed successfully
400 Bad Request Missing required parameters
401 Unauthorized Invalid API key
404 Not Found Agent or resource not found
500 Server Error Internal server error
Example Error Response
{ "error": "Missing required fields", "message": "agent_id, from_number, and to_number are required", "timestamp": "2026-04-18T18:05:00.000Z" }
